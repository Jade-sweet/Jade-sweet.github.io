安装PyMySQL三方库：
1.从文件设置里面安装第三方库。

PyCharm —> (File—>)Settings/Preferences
—> Project Interpreter —> + —> Install Package

2.从控制台安装库

Terminal —> pip install pymysql -i https://pypi.doubanio.com/simple

生成依赖项清单文件 - requirements.txt这里的txt文件名必须为requirements.txt

根据requirements文件重建项目依赖项

pip install -r requirements.txt

以下是常见的连接错误类型：

错误：连接被对端重置 - 阿里云封IP - 安全管控中添加IP白名单
Connection reset by peer

错误：无法连接指定的服务器 - IP地址或者端口写错
“Can’t connect to MySQL server on ‘1.2.3.4’ (timed out)”

错误：访问被拒绝 - 用户名或密码错误
“Access denied for user ‘root’@’1.2.3.4’ (using password: YES)”)

错误：未知的数据库 - 数据库名字写错了
“Unknown database ‘…’”

错误：AttributeError - 字符集写错了
AttributeError: ‘NoneType’ object has no attribute ‘encoding’

附

在日常的开发过程中，我们应该养成版本控制的习惯，在pycharm中要做版本控制，大概有以下几个步骤：

①现在第三方平台新建一个仓库来实现版本的控制

②复制仓库的SSH网址或者HTTPS网址，打开pycharm

③到初始界面选择check out from Version Control下的Git

④将复制的网址粘贴上去并进行test，检查是否能连通，测试通过点Clone

⑤Clone下来的文件是没有虚拟环境的，在setting下寻找Project Interpreter上的齿轮进行ADD操作或者选用已有的虚拟环境的操作，add操作是在本文件下建立一个venv文件

⑥创建requirements.txt文件来描述库的使用情况，方便别人使用我们的文件代码时直到应该安装哪些支持库，这里需用到重定向操作来写入我们安装的库，在控制台输入 pip list > requirements.txt，这样我们就将安装的库文件写入到requirements文件中了

⑦想批次安装库文件，直接在控制台输入 pip install -r requirements.txt 即可

⑧版本控制，点左上角的√，选中需上传的文件即可，
注意：venv文件不能上传，requirements.txt文件要上传
想要不上传的文件隐藏，只需在文件夹中新建一个普通的file文件，名字为.gitignore，里面写上你需要隐藏的文件名即可
⑨上传到第三方平台，选择上方的vcs里面的git下面的push即可
⑩从第三方平台同步数据，选择上方的vcs里面的git下面的pull即可
